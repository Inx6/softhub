# Softhub

## 一、注册、登录
### 1.注册
```
    接口:http://101.35.82.220:666/add_u

    类型:POST

    请求参数: 
     nick: String,      // 昵称
     name: String,      // 账号名
     pwd: String,       // 密码
     avatar: String,    // 头像 (默认为后台设置)
     level: String,     // 等级 (默认为0)
     status: i32,       // 用户状态 (正常0、封禁1，默认为正常)
     time: String       // 注册时间
```
```
    请求成功：
        "成功注册啦！"
    
    请求失败：
        "不晓得什么原因，注册出错了......"
```
### 2.登录
```
    接口:http://101.35.82.220:666/login

    类型:GET

    请求参数: 
     name: String,         // 账号
     pwd: String           // 密码
```
```
    请求成功：

        "suc": [
            [
                "昵称",
                "账号",
                "头像地址",

            "data": "密码错误啦！"
```

## 四、上传
### 1.软件上传
#### （1）链接上传
链接上传是指不上传应用/apk/exe等文件，只选择上传一个链接地址，这个地址可以是网盘地址，也可以是该软件的下载地址。（无论怎么讲，上传地址对应用上传，优势在我！)<br/>
**!!! 注 意: 应用上传后依旧需要提交此环节项 !!!**
```
    接口:http://101.35.82.220:666/add_url

    类型:POST

    请求参数: 
     name: String,      // 名称
     url: String,       // 下载地址
     ic: String,        // 图标地址
     ty: String,        // 一级分类划分区
     label: String,     // 二级分类划分区
     version: String,   // 应用简介，作用
     package: String    // 应用包名
```
```
    请求成功：
        上传成功：
            "上传成功啦！"
        上传失败：
            "不晓得什么原因，上传出错了......"
    请求失败:
        404
```
#### （2）应用上传
```
    接口:http://101.35.82.220:667/

    类型:POST

    请求参数:
     file: 文件,
     type: "apk",       //固定为apk
     name: 文件名       
```
```
    请求成功：
        上传成功：
            {
                "上传成功啦！地址为": "http://127.0.0.1:251/apks/3VPN-release.apk"
            }
        上传失败:
        1.{
            "不晓得什么原因，上传失败啦(°ー°〃) "
        }
        2.{
            "上传失败,文件库里面已经有你的上传地址了,改下文件名试试？(°ー°〃) "
        }
    请求失败:
        404
```
